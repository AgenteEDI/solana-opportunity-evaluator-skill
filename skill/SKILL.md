# Solana Opportunity Evaluator

Use this skill when the user needs to decide whether a Solana opportunity deserves time, money, roadmap focus, or reputation.

## What This Skill Is For

- grants and bounty evaluation
- partnership and BD triage
- integration timing decisions
- accelerator, incubator, or hackathon selection
- sponsor or vendor opportunity review
- contributor or contractor opportunity screening

## Default Operating Procedure

1. Read the source material first: listing, docs, repo, public comms, forms, and constraints.
2. Classify the opportunity: bounty, grant, integration, partnership, hiring, vendor, or mixed.
3. Run the evaluation workflow in [opportunity-eval.md](opportunity-eval.md).
4. Score it with [scoring.md](scoring.md).
5. Deliver one of the output artifacts from [artifacts.md](artifacts.md).

## Decision Standard

The goal is not "sound smart". The goal is to answer:

- should we spend time on this now
- what could go right
- what could go wrong
- what evidence is still missing
- what the next move should be

## Read These Files When Needed

- [opportunity-eval.md](opportunity-eval.md): full workflow
- [scoring.md](scoring.md): scoring dimensions and rating rules
- [artifacts.md](artifacts.md): memo and recommendation templates
